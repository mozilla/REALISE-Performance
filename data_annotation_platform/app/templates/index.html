General Introduction


Hello, This project is done by student Mohamed Bilel Besbes and Professor Diego Elias Damasceno Costa from REALISE Lab at Concordia University in Montréal, Canada.


Main Contribution


  • General overview


    This project aims to optimize the performance engineering workflow at Mozilla, specifically the part of automatic detection by making it detect anomalies (regressions and improvements) faster and more precisely, thus saving investigation time for Mozilla performance engineers. The aim to do this by exploring alternatives to the existing method in use (Student T-test).

  • Current Progress Status


    We collected data from Mozilla systems earlier, we explored Mozilla’s performance engineering workflow for insights, and we published the collected data and explored insights in a paper that we presented in ICPE’25.

  • Future Directions


    As we explored the data, it was hard to have a baseline to compare Mozilla’s current Change Point Detection method (Student T-test). BUT, the issue is that the current data of the signatures timeseries contains backfills and retriggers, making it hard to replicate the original time series. Therefore, we wanted to make a dataset of experts-validated annotation to use it as a ground truth.
