## Context

This GitHub repository contains work done to investigate the use of change point detection (CPD) methods to catch performance anomalies in Mozilla workflows. The data associated with this project can be found [here](https://doi.org/10.5281/zenodo.20382322).

## Repository Breakdown

The work has revolved around extracting data from Mozilla systems, collecting annotation data from Mozilla experts, and evaluating different change point detection techniques to detect anomalies in Mozilla data. This work is organized in the following folders: `data_annotation_platform`, `data_extraction_transformation`, `prediction_generation`, and `prediction_postprocessing_scripts`.

**Every subfolder has its own `README.md` file with a breakdown of its contents. Here is a quick overview of each folder.**

## `data_annotation_platform` Overview

The `data_annotation_platform` folder contains the code for the data annotation platform used to collect ground truth for assessing the CPD methods.

## `data_extraction_transformation` Overview

The `data_extraction_transformation` folder contains scripts and notebooks to extract performance-related data from the [Treeherder API](https://treeherder.mozilla.org/docs/), transform them, and perform preliminary analysis on them.

## `prediction_generation` Overview

The `prediction_generation` folder contains scripts to detect change points in the data using [TCPDBench](https://github.com/SimonEismann/TCPDBench). The TCPDBench project has been extended and adapted to fit the use case of this project by adding several CPD method implementations.

## `prediction_postprocessing_scripts` Overview

The `prediction_postprocessing_scripts` folder contains scripts to further transform and analyze the data generated by TCPDBench — for example, by combining the results of several CPD methods. The code related to the voting strategies used in this project is also located in this folder.
